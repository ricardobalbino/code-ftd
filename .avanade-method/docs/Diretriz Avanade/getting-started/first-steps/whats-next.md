# What's Next?

After having done these first steps above, you are already able to customize Dataverse solutions in Customization Master and deploy them through all environments and basically run your project from implementation perspective. Nevertheless, there is more you might want to do or could explore to do with the help of the **BizApps Core Accelerator**.

### Add Code

If you are in light-weight mode, you can enable frontend and backend code as an opt-in. Otherwise, the respective capabilities already exist in your setup and you can right away start working with the rich code acceleration possibilities:

* **Backend**: Start [here](../backend/visual-studio-solution.md) to understand the structure and to dive into the examples for the development of plug-ins, Custom APIs etc. For in-depth understanding of backend code development guidelines, see [here](../../../development/Backend/).

* **Frontend**: Start [here](../frontend/frontend-structure.md) to get to know the structure and examples for web resource, form script and ribbon control development. For development guidelines, see [here](../../../development/Frontend).

* **PCF**: See [here](../next-steps/pcf-development.md) on how to add and develop PCF components.

### Add Configuration and Test Data

It is quite common and recommended to manage all data which is not coming from manual data entry or interfaces, as part of the repository and get it deployed automatically. [Data Management](../next-steps/data-management.md) holds insights and directions on this. The example contained in the repository are easy to enabled and to adapt.

### Add Automation Steps

Another important and recommended part is the automation of certain deployment steps which can be executed before and after the solution import as explained in [Deployment Automation](../next-steps/deployment-automation.md).

**Examples:**

* Set the root business unit
* Update System Settings
* Assign roles to application users (for integration mostly)
* Service Bus Endpoint registration
* Update Themes based on the target environment (different navigation bar color)
* ...

### Advanced Scenarios

See the other pages under [Next Steps](../next-steps/index.md) for other scenarios which could come up over time (for examples to manage the local development environments, resync lower Dataverse environments, get updates for BCA components, etc.) and how they are supported by the BCA already or in the future.
