## 🚀 Objetivo | Purpose

Ensinar o Copilot Agent a:

1. **Analisar** o workspace atual, identificar tecnologias.
2. **Gerar/atualizar** arquivos em pastas corretas:

   * **Instruções globais:** `.github/copilot-instructions.md`
   * **Instruções específicas:** `.github/instructions/*.instructions.md`
   * **Prompts reutilizáveis:** `.github/prompts/*.prompt.md`
3. **Garantir existência** das pastas corretas (via comando).
4. **Editar corretamente** `settings.json` do workspace.
5. **Confirmar** claramente antes de realizar qualquer alteração.

---

## ⚙️ Ajustes obrigatórios em `settings.json`

Sempre use essa estrutura:

```jsonc
{
  // Ativa prompt e instruções no projeto
  "chat.promptFiles": true,
  "github.copilot.chat.codeGeneration.useInstructionFiles": true,

  // Localizações dos arquivos instrução e prompts
  "chat.instructionsFilesLocations": {".github/instructions": true},
  "chat.promptFilesLocations": {".github/prompts": true},

}
```

> Caso não exista, crie em `.vscode/settings.json`.

---

## 📋 Fluxo claro de execução:

1. Use o `codebase` para detectar linguagens e frameworks.
2. Confirme com usuário as tecnologias detectadas.
3. Garanta existência das pastas:

```powershell
New-Item -ItemType Directory -Force -Path .github/instructions, .github/prompts, .vscode
```

4. Crie arquivos seguindo templates abaixo.
5. Exiba alterações antes e confirme claramente com usuário.
6. Somente após confirmação, aplique mudanças.

## 📚 Templates rápidos

### Arquivo global (`.github/copilot-instructions.md`)

```markdown
# Padrões Gerais do Projeto

- (Inserir regras gerais)
- Sempre adicione comentário "Generated by Copilot" ao gerar código
```

### Instruções específicas (`.github/instructions/react.instructions.md`)

```markdown
---
applyTo: "**/*.tsx,**/*.ts"
---
# React Regras

- Componentes funcionais com Hooks
- Estrutura do projeto com Next.js
```

### Prompt reutilizável (`.github/prompts/new-api.prompt.md`)

```markdown
---
mode: 'agent'
description: 'Criar endpoint REST no NestJS'
---
Crie um novo endpoint REST seguindo a arquitetura do projeto.
```
