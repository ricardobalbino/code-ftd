| type                          | update | **Optional:** <ul><li>If no type is specified, a default configuration is assumed. It works for most of the basic types but if you need to configure a specific advanced scenario, add the type and follow the information below.</li><li>decimal: The field value is converted to decimal.</li><li>wholenumber: The field value is converted to inter.</li><li>dateonly: The field value is converted to date and 11 hours are added so that the day is shown correctly in all time zones.</li><li>datetime: The field value is converted to a valid datetime value.</li><li>twooptions: If the uppercase field value is "YES", the target value is true, otherwise it is false.</li><li>optionset: The (English) label of the option set is looked up according to the field value.</li><li>choices: The (English) labels of the choice items (multiple option sets), separated by semicolon (if the label has a semicolon in it, use "\;" to escape it).</li><li>fetchxml: If the field value is a valid fetch XML expression, it is handled accordingly, i.e. GUIDs are resolved dynamically according to the current environment.</li><li>path: Absolute path to a file which shall be attached to an "annotation" entity. The relation to another entity (e.g. case) can be defined as a referenced entity column. Document upload does not work for any other entity. See example in EntityConfiguration.xml.</li><li>securityrole: Assignment of security roles only works for entities with the name "systemuser" or "team". The column name can be anything by must be unique among all columns (e.g. "securityrole1" etc.). The actual security role name can be given in the column header or **optionally** provided in the attribute "typeextension".</li><li>fieldsecurityprofile: Assignment of field security profiles only works for entities with the name "systemuser" or "team". The column name can be anything by must be unique among all columns (e.g. "profile1" etc.). The profile name is handled in the same was as Security Role names</li><li>assign: This type only works with referenced entities (refkey and refentity) of type "team" or "systemuser". It can be used to assign the respective record to the team or user. (If for example the same workbook column has to be used to populate a lookup field to the team and to assign the record to the team, then an additional configuration of type "assign" has to be added for the same column using a different column name.)</li><li>share: With this type, a record can be shared with a given team whereas the correct configuration for refkey and refentity must be provided to look up the according team.</li><li>changetype: Defines the type of action to be performed on the row. The default change type is "UPDATE". Using "NEW" as the value, a force create of the row is performed.</li><li>datetimenow: Sets the configured date time field to the current date. When using this type, the column property needs to be set to "-1". This pattern can for example be used to store the timestamp of the last update done during the import.</li><li>currency: Imports values for fields which are of the type "Money".</li><li>guid: Allows to create or update records with a specific guid which can be used to migrate sets of data 1:1 from one source system to another preserving also relationships between records.</li><li>file: Reads or writes the field content from a file whose path is provided in the column. For export this can be combined with the typeextension attribute which would hold the extension of the exported file (e.g. "txt"). **Note**: This type can help by outsourcing larger column content to files but it is not related to the File type column which is handled separately.</li><li>merge: This type indicates that the record referenced in this column must be merged into the main entity record for the row. It is only valid with name="subordinateid" (property of the respective MergeRequest parameter) and with refentity="<same entity as defined for the entity configuration>" as well as refkey provided to identify the merging subordinate record (see 0).</li></ul> |
