# app.config

The following settings can be found and modified in the `app.config` and/or overruled by command-line switches:

| Setting                                   | Windows Form | Command-Line | Description                                                                                                                                                                                                                                                                                    |
| ----------------------------------------- | ------------ | ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ConnectionString                          | Yes          | Yes          | In the app.config you can define a Dataverse connection string. See above how to specify the connection via the command-line.                                                                                                                                                                        |
| DefaultTemplateLocation                   | Yes          | Yes          | The default Excel template can be overruled by the command-line.                                                                                                                                                                                                                               |
| DefaultConfiguration                      | Yes          | Yes          | The default configuration XML can be overruled by the command-line. It must be in the _EntityXml_ subfolder like all configurations.                                                                                                                                                           |
| CacheSize                                 | Yes          | No           | The cache size can be used to optimize the number of records which are created or updated in chunks. It was found that a value of 50 provides good results in terms of speed and stability. As soon as an entity configuration is self-referencing, the cache size is reduced to 1 at runtime. |
| ServersToAllowCleanUp                     | Yes          | Yes          | Comma-separated list of URLs for which a clean-up/deletion of entity data is explicitly allowed. The clean-up only works in combination with the command-line setting described above.                                                                                                         |
| LanguageCode\_\*                          | Yes          | No           | Language codes (e.g. for email templates) must be explicitly configured in case the “languagecode” type/column is used. Missing language codes must be added manually to the _app.config_ file.                                                                                                |
| LogFileSeparator                          | Yes          | No           | See below.                                                                                                                                                                                                                                                                                     |
| StringReplacements                        | No           | Yes          | Replaces search expressions with specific values (this can be used to dynamically set values based on the target environment): <search1|replace1;search2|replace2;...>                                                                                                                         |
| UseAssociateRequestForAddingUsersToTeams  | Yes          | Yes          | Can be set to “true” in order to avoid using the standard AddTeamMembers and RemoveTeamMembers requests for Team management.                                                                                                                                                                   |
| IgnoreExistingRolesForSecurityRoleUpdates | Yes          | Yes          | Can be set to “true” in order to speed up the import of Security Roles by ignoring the check of existing user roles. In case the role assignment already exists, this will result in a “duplicate key” error for each assignment which must be ignored then.                                   |
