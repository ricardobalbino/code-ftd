# BCA vs. Microsoft Power Platform Build Tools

BCA follows the principle to embrace all Microsoft recommendations and toolings. The usage of [Power Platform Build Tools](https://docs.microsoft.com/en-us/power-platform/alm/devops-build-tools) is not possible yet because of various limitations which are outlined in this comparison.

!!! info
    In case you have any suggestion how to overcome the Build Tools limitation and incorporate them into the BCA tooling, please [reach out and leave a feedback](https://platform.avanade.com/support/tickets).

| **#** | **Features**                                                                                                                                       | **BCA** | **Build Tools** | **Background**                                                                                                                                                                             |
| :---: | -------------------------------------------------------------------------------------------------------------------------------------------------- | :-----: | :-------------: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **1** | Setup without Azure DevOps Organization approval (Azure DevOps Extension)                                                                          |    X    |        -        | Build Tools require the installation of an extension from the Azure DevOps Marketplace which is sometimes not allowed or appreciated by IT departments. BCA runs without any installation. |
| **2** | Use client-owned Azure Key Vault to store Dataverse credentials (including optional VNET)                                                          |    X    |        -        | Some clients prefer to use a Key Vault which is managed by them which is possible with the BCA and not with the Build Tools as those are using Service Connections.                        |
| **3** | Certificate-based authentication against Dataverse                                                                                                 |    X    |        -        | Some clients require certificate-based authentication which is supported by Dataverse Tooling used by BCA but not with the Service Connections required by the Build Tools.                |
| **4** | Use same tooling across local CLI and Azure DevOps pipelines                                                                                       |    X    |        -        | Build Tools can only be used in pipelines and not in local scripts (which is necessary for the BCA in order to fully support developer and customizer workflows)                           |
| **5** | Import data through pipelines without [MS tooling limitations](../tools/Entity-Management-Cockpit/Comparison-with-Configuration-Migration-Tool.md) |    X    |        -        |
| **6** | Execute typical [Dataverse deployment tasks](../getting-started/next-steps/deployment-automation.md) (set root business unit, consistenly configure system settings, ...)  |    X    |        -        |
| **...** | ...  |       |                |
